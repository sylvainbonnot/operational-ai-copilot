# Architecture

## Overview

The system is a retrieval-augmented generation (RAG) API with an intent-driven agent layer on top. Every component is independently testable and replaceable.

## Layers

### 1. Data layer (`data/`)

Synthetic industrial data generated by `data/generate.py` (deterministic, `--seed 42`).

| File | Contents |
|------|----------|
| `machines.json` | 12 machines across 3 sites |
| `tickets.jsonl` | 120 incident tickets |
| `sensor_readings.jsonl` | ~12k sensor readings with anomaly flags |
| `anomaly_summaries.jsonl` | 30-day windowed anomaly aggregates |
| `manual_sections.json` | 20 equipment manual sections |
| `operator_notes.jsonl` | 60 operator shift notes |

### 2. Storage layer (`app/storage/`)

**`database.py`** — async SQLAlchemy engine, `init_db()` creates:
- `chunks` table: `id, source_type, source_id, machine_id, site, content, metadata JSONB, embedding vector(768)`
- `feedback` table: persists human ratings with promotion flag

**`vector_store.py`** — `embed_texts()` via Ollama `/api/embed`, `upsert_chunks()` with ON CONFLICT, `similarity_search()` using cosine distance (`<=>` operator). IVFFlat index with 50 lists.

**`feedback_store.py`** — `save_feedback()`, `list_feedback()`, `mark_promoted()`.

### 3. Ingestion layer (`app/ingestion/`)

**`chunking.py`** — per-type chunking strategies:
- Manuals: one chunk per section, title + metadata prepended
- Tickets: all structured fields serialised as readable prose
- Operator notes: note text + date + operator
- Anomaly summaries: window stats + description

**`run_all.py`** — orchestrates all ingestors sequentially. Idempotent (upsert on conflict).

### 4. RAG layer (`app/rag/`)

**`retriever.py`** — hybrid retrieval: always runs unscoped search (catches manuals with `machine_id=NULL`), merges with machine-scoped search when `machine_id` is provided, deduplicates by chunk_id, re-ranks by score.

**`prompts.py`** — system prompt enforces citation and refusal behaviour. User prompt injects numbered context chunks.

**`answer_generator.py`** — calls Ollama `/api/chat` with `temperature=0.1` for factual answers. Returns `GenerationResult` with token counts and latency.

**`grounding.py`** — heuristic grounding: counts source IDs cited in answer / total sources. Explicit refusal patterns count as grounded. `groundedness_threshold` from settings.

### 5. Agent layer (`app/agents/`)

**`intent.py`** — keyword regex classifier. 6 intents: `diagnosis`, `summary`, `similar_incidents`, `maintenance_recommendation`, `risk_assessment`, `unknown`. No LLM call — deterministic and fast.

**`tools.py`** — 5 async tool functions, each tracking calls via Prometheus counter:
- `tool_retrieve_incidents` — filtered by source_type=ticket
- `tool_retrieve_manuals` — filtered by source_type=manual
- `tool_retrieve_anomalies` — filtered by source_type=anomaly_summary
- `tool_get_machine_history` — operator notes for a machine
- `tool_get_machine_profile` — static metadata from machines.json

**`graph.py`** — orchestration loop. Per-intent tool selection → context assembly → deduplication → generation → grounding check. No framework dependency.

### 6. API layer (`app/api/`)

FastAPI routers:

| Route | Handler |
|-------|---------|
| `POST /ask` | Full agent pipeline |
| `POST /ask/incident/summarize` | Ticket summary via agent |
| `POST /ask/incident/diagnose` | Symptom diagnosis via agent |
| `POST /feedback` | Submit human feedback |
| `GET /feedback` | List feedback (filterable) |
| `POST /feedback/{id}/promote` | Flag for golden dataset |
| `POST /eval/run` | Trigger eval run |
| `GET /eval/reports` | List report files |
| `GET /health` | Health check |
| `GET /metrics` | Prometheus metrics |

### 7. Evaluation layer (`app/evaluation/`)

**`metrics.py`** — `precision_at_k`, `recall_at_k`, `evidence_hit_rate`, `fact_coverage`, `refusal_correct`, `compute_pass`. No LLM required.

**`eval_runner.py`** — loads `eval/golden_questions.jsonl`, runs each question through the full pipeline (retrieve → generate), scores results, pushes summary gauges to Prometheus, writes JSON + Markdown report. CI gate: `pass_rate >= 60%`, `mean_hit_rate >= 0.50`.

### 8. Observability layer

**`app/core/telemetry.py`** — all Prometheus metric objects defined centrally.

**`app/core/middleware.py`** — `PrometheusMiddleware` instruments all tracked POST endpoints.

**`ops/prometheus/`** — `prometheus.yml` scrapes `/metrics` every 15s; `alerts.yml` defines error rate, latency, and groundedness alerts.

**`ops/dashboards/copilot.json`** — auto-provisioned Grafana dashboard with 4 rows: Request Health, RAG Quality, LLM Usage, Agent Tools.

## Key design decisions

| Decision | Rationale |
|----------|-----------|
| Custom agent over LangGraph | No framework coupling; every step is a plain async function, trivially testable |
| Hybrid retrieval (scoped + unscoped) | Manuals have `machine_id=NULL`; pure scoped search misses them |
| `evidence_hit_rate` as primary eval metric | With top_k=8 and 1–2 expected docs, raw precision@k is misleadingly low |
| `temperature=0.1` for generation | Factual operational answers need low variance |
| IVFFlat with lists=50 | Appropriate for ~250 vectors; switch to HNSW at 10k+ |
| Feedback → golden promotion | Closes the human-in-the-loop cycle; bad answers become regression tests |
